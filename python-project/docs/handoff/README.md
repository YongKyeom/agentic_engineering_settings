# Handoff Docs

이 폴더는 세션 인수인계 메모를 로컬에 남기기 위한 공간입니다.
Handoff는 다른 Agent나 사용자에게 넘길 수 있는 최소 전달 패키지입니다.
작업이 길어져서 새 Codex 세션으로 넘겨야 하거나, Claude와 이어서 작업해야 하거나, 다음 날 다시 이어서 개발해야 할 때 사용합니다.

이 폴더도 `plan/`과 같은 기준을 따릅니다.
폴더 구조는 Git에 남기지만, 개별 handoff 파일은 기본적으로 commit하지 않습니다.

## 사용 목적

- 긴 세션의 현재 상태를 짧게 보존합니다.
- 다음 세션이 전체 대화 기록을 다시 읽지 않고 이어서 작업하게 합니다.
- 변경한 파일, 실행한 명령, 결정한 사항, 남은 위험을 놓치지 않게 합니다.
- 사람, Codex, Claude 사이에 작업 맥락을 넘깁니다.

## Git 추적 기준

- 이 폴더 자체는 Git에 유지합니다.
- `.gitkeep`, `README.md`, `AGENTS.md`는 commit합니다.
- 개별 handoff 파일은 기본적으로 commit하지 않습니다.
- 사용자가 명시적으로 공유 문서로 남기라고 했을 때만 handoff 파일을 commit합니다.
- 장기적으로 보존해야 하는 기술 결정은 `docs/decisions/`로 옮깁니다.

## Handoff Template

```md
# Handoff: Title

## 상태

- 완료 / 진행 중 / 보류

## 작업 범위

- 이번 handoff가 다루는 범위를 한두 줄로 적습니다.

## 변경 파일

- `path/to/file` - 변경 요약

## 핵심 변경 내용

- 무엇을 왜 바꿨는지 2~5개 이내로 적습니다.

## 주요 결정사항

- 다음 작업자가 결정을 뒤집지 않도록 결정과 근거를 짧게 적습니다.

## 검증

- 실행한 명령: ...
- 수동 확인: ...
- 검증하지 못한 항목: ...

## 핵심 파일

- `path/to/file` - 다음 작업자가 먼저 봐야 하는 이유

## 주의사항

- known gotcha, 우회 방법, 현재 가정, 문서와 코드의 경계를 적습니다.

## 남은 리스크

- 바로 문제가 될 수 있는 항목만 적습니다.

## 다음 액션

- [ ] 다음 작업자가 바로 실행할 수 있는 첫 단계
- [ ] 다음 작업 2

## 이전 handoff 참조

- 없음 / `YYYYMMDD_<topic>.md`

## 로컬 산출물 처리 여부

- 생성 없음 / 로컬만 생성 / 커밋함
```

## 작성 기준

- 새 세션이 바로 이어서 작업할 수 있을 만큼 구체적으로 씁니다.
- 실행하지 않은 검증을 실행했다고 쓰지 않습니다.
- 더 최신 handoff가 생기면 오래된 handoff는 stale로 봅니다.
- 불필요하게 긴 로그를 붙이지 말고, 핵심 결과만 요약합니다.
- 파일로 남길 때 권장 위치는 `docs/handoff/YYYYMMDD_<주제>.md`입니다.
